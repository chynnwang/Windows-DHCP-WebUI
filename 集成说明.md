# DHCP 管理模块 — 集成说明

本模块是一个用于集中管理多台 Windows Server 上 DHCP 服务(作用域、租约、保留、选项、
工区分组、租约日志、飞书告警)的 Web 应用,内置**登录鉴权与两级角色(Admin / Viewer)**。
既可独立部署直接访问,也可用 iframe 嵌入上层平台。

> 文中出现的 `dhcp.example.com:8090` 仅为示例地址,请替换为你自己的部署地址。

---

## 1. 快速集成(iframe 嵌入)

在你的平台里用 iframe 嵌入即可:

```html
<iframe
  src="http://dhcp.example.com:8090/?theme=dark"
  style="width:100%;height:100%;border:0"
  allow="clipboard-write"
></iframe>
```

- 前端是单页应用(SPA),使用 **hash 路由**(地址形如 `/#/manage`),嵌入时无需为子路由做额外配置。
- 服务端**未设置** `X-Frame-Options` / `CSP frame-ancestors`,可被任意父页面 iframe 嵌入(内网可信假设下)。
- `allow="clipboard-write"`:让「接入服务器」弹窗里的「复制安装命令」按钮可用(HTTP 非安全上下文已内置降级方案,不加也能用)。
- 无论是否嵌入,首次访问都会跳转登录页,需先登录(见 §3)。

---

## 2. 主题联动(深色 / 浅色)

模块支持浅色与深色两套主题,优先级:**URL 参数 > localStorage > 默认浅色**。

| 方式 | 用法 | 说明 |
|------|------|------|
| URL 参数 | `?theme=dark` 或 `?theme=light` | 首次加载时生效,最简单 |
| postMessage | 父页面向 iframe 发送消息 | 运行时实时切换,随父平台主题联动 |
| 页面内按钮 | 右上角太阳/月亮按钮 | 用户手动切换,存 localStorage(键 `dhcp-theme`) |

父平台运行时切换主题(推荐):

```js
const iframe = document.querySelector('iframe')
iframe.contentWindow.postMessage({ type: 'set-theme', theme: 'dark' }, '*')
// theme 取值:'dark' | 'light'
```

## 3. 登录与鉴权(重要)

- 模块内置登录鉴权,基于 **JWT**。前端登录后拿到 token,后续 API 调用带
  `Authorization: Bearer <token>`;token 过期或无效返回 `401`,前端会自动跳回登录页。
- **两级角色**:
  - **Admin**:全部权限,包括用户管理、所有 DHCP / 告警 / 日志的写操作、Agent 接入与密钥管理。
  - **Viewer**:只读。可查看所有数据,但所有写操作(增删改)会被后端拒绝(`403`),前端也会隐藏对应按钮。
- **默认管理员**:首次启动会自动创建管理员账号 `admin`,初始密码 `admin`。**请登录后尽快在「用户管理」中修改密码或新建你自己的账号**。
- **公开接口**:仅 `GET /api/ping`(存活探测)与 Agent 用的 SignalR hub(`/hubs/agent`,靠连接密钥校验,见 §5)无需登录,其余接口均需 JWT。
- 审计:后端记录操作日志,登录后写操作会记录真实用户名。

## 4. API 概览

Base URL:`http://dhcp.example.com:8090/api`(或后端 `http://127.0.0.1:5280/api`)。返回 JSON。
除 `GET /ping` 外均需 `Authorization: Bearer <token>`;标注「Admin」的接口仅管理员可调用。

| 分类 | 方法与路径 | 说明 |
|------|-----------|------|
| 健康 | `GET /ping` | 存活探测(公开) |
| 鉴权 | `POST /auth/login` | 登录换取 JWT(公开) |
| 鉴权 | `GET /auth/me` | 当前登录用户信息 |
| 鉴权 | `POST /auth/change-password` | 修改自己的密码 |
| 用户 | `GET/POST /users`,`PUT/DELETE /users/{id}` | 用户管理(**Admin**) |
| 工区 | `GET /sites` / `POST /sites` / `PUT /sites/{id}` / `DELETE /sites/{id}` | 工区增删改查(写操作 **Admin**) |
| 服务器 | `GET /servers` / `GET /servers/{id}` | 已纳管的 DHCP 服务器列表 |
| 服务器 | `PUT /servers/{id}` | 重命名(**Admin**) |
| 服务器 | `PUT /servers/{id}/site` | 归入工区(body `{siteId: number\|null}`,**Admin**) |
| 服务器 | `PUT /servers/{id}/callback-url` | 改单台**已连** Agent 的回连地址(body `{url}`);Agent 更新配置后自动重启并按新地址重连(需在线,**Admin**) |
| 服务器 | `DELETE /servers/{id}` | 删除;在线时先远程卸载 Agent 再移除记录(**Admin**) |
| 服务器 | `POST /servers/{id}/health` | 健康检查 |
| 作用域 | `GET/POST /servers/{id}/scopes`,`PUT/DELETE /servers/{id}/scopes/{scopeId}` | 作用域 CRUD(写操作 **Admin**) |
| 作用域 | `GET /servers/{id}/scopes/{scopeId}/statistics` | 使用率统计 |
| 租约 | `GET /servers/{id}/scopes/{scopeId}/leases` | 租约列表 |
| 保留 | `GET/POST /servers/{id}/scopes/{scopeId}/reservations`,`DELETE ...?ip=` | 静态绑定(写操作 **Admin**) |
| 选项 | `GET/PUT /servers/{id}/options`,`DELETE /servers/{id}/options/{optionId}` | 网关/DNS 等(写操作 **Admin**) |
| 租约日志 | `GET /lease-logs?q=&serverId=&page=&pageSize=` | 历史租约检索(分页) |
| 告警 | `GET/PUT /alerts/config`,`POST /alerts/test` | 飞书告警配置与测试(写操作 **Admin**) |
| Agent 接入 | `GET /agent/setup-info` | 返回下载地址、平台地址与预填安装命令(**Admin**) |
| Agent 接入 | `PUT /agent/platform-url` | 设置平台对外地址(body `{url}`,留空恢复自动识别,**Admin**) |
| Agent 接入 | `PUT /agent/enrollment-secret` | 设置 Agent 连接密钥(body `{secret}`,留空恢复默认,**Admin**) |

## 5. 纳管一台新的 DHCP 服务器(Agent 安装)

1. 以管理员登录后点「接入服务器」,或调用 `GET /api/agent/setup-info` 获取:
   - `downloadUrl`:`http://dhcp.example.com:8090/download/DhcpAgent.exe`
   - `platformUrl`:安装命令里 `--server` 用的平台地址(可在弹窗手动改,见 §6)
   - `enrollmentSecret`:Agent 接入本平台的校验密钥(可在弹窗手动改)
   - `installCommand`:已预填平台地址与连接密钥的安装命令
2. 把 `DhcpAgent.exe` 拷到目标 Windows DHCP 服务器,以**管理员** PowerShell 运行安装命令。
3. Agent 作为 Windows 服务运行,主动向平台建立 SignalR 长连接、**自动注册**并出现在列表中
   (默认归入「未分组」)。断线会无限重试重连。
4. 前置条件:运行账号需在 **DHCP Administrators** 组,且服务器能访问平台地址。

> 平台通过 SignalR 向 Agent 下发 PowerShell(DHCP cmdlet)执行,Agent 只出站连接,无需在被管机开入站端口。

**连接密钥(Enrollment Secret)**:所有 Agent 用同一密钥接入,默认 `admin`(后端 `appsettings.json` 的
`Agent:EnrollmentSecret`,可被数据库中的覆盖值优先生效)。在「接入服务器」弹窗点「更改密钥」即可修改,
安装命令会立即按新密钥更新。**注意**:改动后已安装的旧 Agent 会因密钥不符而无法重连,需用新密钥重装,请谨慎修改。

## 6. 平台地址 / Agent 回连地址的维护

Agent 通过配置里的 `ServerUrl` 回连平台。平台迁移、换域名或换 IP 时,用以下两处维护地址:

**A. 新装 Agent —「接入服务器」弹窗 · 平台地址**
弹窗顶部显示当前「平台地址」并标注**自动识别 / 手动设置**:
- 默认自动识别为你访问平台用的地址(`Request.Host`)。若从内网 IP、`127.0.0.1` 或被 iframe 嵌入时识别不准,点「更改地址」填入正确的对外地址(域名或 IP,含端口)。
- 保存后下载链接与安装命令**立即**按新地址生成,供新装 Agent 使用;点「恢复自动」清除手动设置。
- 对应接口:`PUT /api/agent/platform-url`,body `{ "url": "http://dhcp.example.com:8090" }`(留空恢复自动)。存于后端数据库,即时生效,无需重启。

**B. 已连 Agent —服务器「更多」·「更改回连地址」**
针对**已经连上**的单台服务器:在「工区 / 服务器」页该服务器的「更多」菜单点「更改回连地址」,输入新地址。平台经 SignalR 通知该 Agent 更新本地配置并**自动重启**、按新地址重连(仅在线时可下发)。
- 对应接口:`PUT /api/servers/{id}/callback-url`,body `{ "url": "..." }`。

> 迁移标准流程:搬家前(Agent 仍连在旧平台)逐台用 **B** 把地址改到新地址 → 迁移平台 → Agent 自动重连过去;同时用 **A** 把弹窗平台地址改成新地址,供之后新装的 Agent 使用。
> **最省事**的办法是给平台绑一个**固定域名**,以后搬家只改 DNS,A/B 都不用动。

## 7. 部署信息(运维参考)

以 Linux + Nginx 反代为例的一种参考部署:

| 项 | 值 |
|----|----|
| 访问端口 | Nginx `:8090` → 后端 `127.0.0.1:5280` |
| 后端 | systemd 服务 `dhcpweb`,自包含单文件 `/opt/dhcpweb/DhcpWeb.Api` |
| 前端 | 静态资源 `/var/www/dhcpweb` |
| 数据库 | SQLite `/opt/dhcpweb/dhcpweb.db` |
| Agent 下载路由 | Nginx `location = /download/DhcpAgent.exe` |

生产环境请在 `appsettings.Production.json` 中配置强随机的 `Jwt:Key`(见 README)。

常用运维命令(服务器上):

```bash
sudo systemctl status dhcpweb      # 查看状态
sudo systemctl restart dhcpweb     # 重启
curl -s http://127.0.0.1:5280/api/ping   # 存活探测
```
